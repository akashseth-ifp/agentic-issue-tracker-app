### Commit 2: Add Async Database and Settings Configuration

**Install packages first:**
```bash
uv add fastapi uvicorn[standard] pydantic-settings
```

- **`fastapi`** — the web framework. Handles routing, request parsing, response serialization, and dependency injection. Node.js analog: Express.js.
- **`uvicorn[standard]`** — the ASGI server that actually runs FastAPI. `[standard]` adds websocket support and faster event loop (uvloop). Node.js analog: `nodemon` / `node server.js`.
- **`pydantic-settings`** — reads `.env` files into a typed Python class. If a required field is missing from `.env`, the app crashes at startup with a clear error instead of a silent `undefined`. Node.js analog: `dotenv` + Zod.

**Also install the async DB stack:**
```bash
uv add sqlalchemy aiosqlite
```

- **`sqlalchemy`** — the ORM. Maps Python classes to database tables and generates SQL. The `2.x` version includes `sqlalchemy.ext.asyncio` for fully async DB access. Node.js analog: Prisma / TypeORM.
- **`aiosqlite`** — async driver for SQLite. SQLAlchemy's async engine needs a non-blocking driver underneath; `aiosqlite` wraps the stdlib `sqlite3` module to be `await`-able. Without it, `create_async_engine("sqlite+aiosqlite://...")` fails at import time.

**Files:** `app/core/config.py`, `app/db/database.py`

**`app/core/config.py`:**
```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    DATABASE_URL: str
    CORS_ORIGINS: list[str] = ["http://localhost:4200"]

    class Config:
        env_file = ".env"

settings = Settings()
```

`pydantic-settings` reads `.env` at import time. If `DATABASE_URL` is missing, the app crashes immediately with a clear error — not a silent `undefined` at runtime. `settings` is a module-level singleton imported wherever config is needed.

**`app/db/database.py`:**
```python
from sqlalchemy.ext.asyncio import (
    create_async_engine,
    AsyncSession,
    async_sessionmaker,
)
from sqlalchemy.orm import DeclarativeBase
from typing import AsyncGenerator
from app.core.config import settings

engine = create_async_engine(
    settings.DATABASE_URL,
    echo=False,       # set True to log SQL in dev
)

AsyncSessionLocal = async_sessionmaker(
    engine,
    expire_on_commit=False,  # objects stay accessible after commit in async context
    class_=AsyncSession,
)

class Base(DeclarativeBase):
    pass

async def get_db() -> AsyncGenerator[AsyncSession, None]:
    async with AsyncSessionLocal() as session:
        yield session
```

**Why `expire_on_commit=False`:** In sync SQLAlchemy, accessing an attribute after `commit()` triggers a lazy-load. In async, that lazy-load would hit an already-closed event loop and raise an error. `expire_on_commit=False` keeps attribute values cached on the object after commit — no extra query needed.

**Why `async with` instead of `try/finally`:** `async with AsyncSessionLocal() as session` is the async context manager pattern. It automatically calls `session.close()` on exit, even if the handler raises. Cleaner than explicit try/finally.

Node.js analog: `engine` = connection pool, `AsyncSessionLocal` = `pool.connect()` factory, `get_db` = middleware that opens a connection and releases it after the request.

**Verify:** `python -c "from app.db.database import engine, get_db; print('ok')"` prints `ok`.

### Commit 3: Define Issue Domain Model and Schemas

**Files:** `app/models/issue.py`, `app/schemas/issue.py`

**`app/models/issue.py`:**
```python
import enum
from sqlalchemy import String, DateTime, Enum as SAEnum
from sqlalchemy.orm import Mapped, mapped_column
from datetime import datetime, timezone
from app.db.database import Base

class IssueStatus(str, enum.Enum):
    OPEN = "Open"
    IN_PROGRESS = "In Progress"
    CLOSED = "Closed"

class Issue(Base):
    __tablename__ = "issues"

    id: Mapped[int] = mapped_column(primary_key=True, index=True)
    title: Mapped[str] = mapped_column(String(255), nullable=False, index=True)
    description: Mapped[str | None] = mapped_column(String(2000), nullable=True)
    status: Mapped[IssueStatus] = mapped_column(
        SAEnum(IssueStatus), default=IssueStatus.OPEN, nullable=False
    )
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        default=lambda: datetime.now(timezone.utc),
        nullable=False,
    )
```

`IssueStatus` inherits from `str` so it serializes as `"Open"` not `"IssueStatus.OPEN"`. `Mapped[int]` is SQLAlchemy 2.0 typed syntax — fully mypy-compatible.

**`app/schemas/issue.py`:**
```python
from pydantic import BaseModel, Field, field_validator
from datetime import datetime
from app.models.issue import IssueStatus

class IssueBase(BaseModel):
    title: str = Field(..., min_length=1, max_length=255)
    description: str | None = Field(None, max_length=2000)
    status: IssueStatus = IssueStatus.OPEN

class IssueCreate(IssueBase):
    @field_validator("title")
    @classmethod
    def title_not_blank(cls, v: str) -> str:
        if not v.strip():
            raise ValueError("title cannot be blank or whitespace only")
        return v.strip()

class IssueUpdate(BaseModel):
    title: str | None = Field(None, min_length=1, max_length=255)
    description: str | None = None
    status: IssueStatus | None = None

class IssueResponse(IssueBase):
    id: int
    created_at: datetime
    model_config = {"from_attributes": True}

class IssuePage(BaseModel):
    items: list[IssueResponse]
    total: int
    page: int
    page_size: int
    total_pages: int
```

`IssuePage` is the paginated list response. `total_pages = ceil(total / page_size)` — computed in the router so the frontend never needs to calculate it. `from_attributes: True` lets Pydantic read SQLAlchemy ORM object attributes directly (was `orm_mode` in Pydantic v1).

**Verify:** `python -c "from app.schemas.issue import IssuePage; print(IssuePage.model_fields.keys())"` lists all fields.

### Commit 4: Add Custom Exception Hierarchy

**File:** `app/core/exceptions.py`

```python
from fastapi import Request
from fastapi.responses import JSONResponse

class AppException(Exception):
    def __init__(self, status_code: int, detail: str):
        self.status_code = status_code
        self.detail = detail

class IssueNotFoundException(AppException):
    def __init__(self, issue_id: int):
        super().__init__(404, f"Issue with id {issue_id} not found")

class InvalidStatusTransitionException(AppException):
    def __init__(self, from_status: str, to_status: str):
        super().__init__(422, f"Cannot transition from '{from_status}' to '{to_status}'")

async def app_exception_handler(request: Request, exc: AppException) -> JSONResponse:
    return JSONResponse(
        status_code=exc.status_code,
        content={"error": exc.detail, "status_code": exc.status_code},
    )
```

The handler is `async def` — FastAPI's exception handling is part of the async request cycle. The service raises `IssueNotFoundException(5)`, and FastAPI intercepts it before it reaches the client, calling this handler to produce the JSON response.

Node.js analog: custom `AppError` class + Express `(err, req, res, next)` error middleware.

**Verify:** `python -c "from app.core.exceptions import IssueNotFoundException; e = IssueNotFoundException(1); print(e.status_code, e.detail)"` prints `404 Issue with id 1 not found`.

### Commit 5: Implement Async Repository Layer

**File:** `app/repositories/issue_repository.py`

```python
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select, func
from app.models.issue import Issue
from app.schemas.issue import IssueCreate, IssueUpdate

class IssueRepository:
    def __init__(self, db: AsyncSession):
        self.db = db

    async def get_all(self, page: int = 1, page_size: int = 20) -> tuple[list[Issue], int]:
        offset = (page - 1) * page_size

        total = await self.db.scalar(
            select(func.count()).select_from(Issue)
        ) or 0

        result = await self.db.execute(
            select(Issue)
            .order_by(Issue.created_at.desc())
            .offset(offset)
            .limit(page_size)
        )
        return list(result.scalars().all()), total

    async def get_by_id(self, issue_id: int) -> Issue | None:
        result = await self.db.execute(
            select(Issue).where(Issue.id == issue_id)
        )
        return result.scalar_one_or_none()

    async def create(self, data: IssueCreate) -> Issue:
        issue = Issue(**data.model_dump())
        self.db.add(issue)
        await self.db.commit()
        await self.db.refresh(issue)
        return issue

    async def update(self, issue: Issue, data: IssueUpdate) -> Issue:
        for field, value in data.model_dump(exclude_unset=True).items():
            setattr(issue, field, value)
        await self.db.commit()
        await self.db.refresh(issue)
        return issue

    async def delete(self, issue: Issue) -> None:
        await self.db.delete(issue)
        await self.db.commit()
```

**`get_all` runs two queries:** one for the total count (needed for pagination metadata), one for the page slice. Both are `await`ed. The count query uses `func.count()` which translates to `SELECT COUNT(*) FROM issues` — a single DB call.

**`model_dump(exclude_unset=True)` on update:** if client sends `{"status": "Closed"}`, only `status` is in the dict. `title` and `description` are left untouched in the DB. Without `exclude_unset=True`, unset fields default to `None` and would null out existing values.

**`scalar_one_or_none()`:** returns the single result or `None` if not found. Never raises — the service layer decides what to do with `None`.

### Commit 6: Implement Service Layer with Business Rules

**File:** `app/services/issue_service.py`

```python
from sqlalchemy.ext.asyncio import AsyncSession
from app.repositories.issue_repository import IssueRepository
from app.core.exceptions import IssueNotFoundException, InvalidStatusTransitionException
from app.models.issue import Issue, IssueStatus
from app.schemas.issue import IssueCreate, IssueUpdate

VALID_TRANSITIONS: dict[IssueStatus, set[IssueStatus]] = {
    IssueStatus.OPEN: {IssueStatus.IN_PROGRESS},
    IssueStatus.IN_PROGRESS: {IssueStatus.CLOSED, IssueStatus.OPEN},
    IssueStatus.CLOSED: set(),  # terminal — no exit
}

class IssueService:
    def __init__(self, db: AsyncSession):
        self.repo = IssueRepository(db)

    async def get_all_issues(self, page: int, page_size: int) -> tuple[list[Issue], int]:
        return await self.repo.get_all(page=page, page_size=page_size)

    async def get_issue_or_raise(self, issue_id: int) -> Issue:
        issue = await self.repo.get_by_id(issue_id)
        if issue is None:
            raise IssueNotFoundException(issue_id)
        return issue

    async def create_issue(self, data: IssueCreate) -> Issue:
        return await self.repo.create(data)

    async def update_issue(self, issue_id: int, data: IssueUpdate) -> Issue:
        issue = await self.get_issue_or_raise(issue_id)
        if data.status is not None and data.status != issue.status:
            allowed = VALID_TRANSITIONS.get(issue.status, set())
            if data.status not in allowed:
                raise InvalidStatusTransitionException(
                    issue.status.value, data.status.value
                )
        return await self.repo.update(issue, data)

    async def delete_issue(self, issue_id: int) -> None:
        issue = await self.get_issue_or_raise(issue_id)
        await self.repo.delete(issue)
```

`VALID_TRANSITIONS` encodes the state machine for issue status. `Closed` maps to an empty set — once closed, no transitions are allowed. This rule exists only here; the router and repository never know about it.

### Commit 7: Add Async API Routes with Pagination + App Entry Point

**Files:** `app/routers/issue_router.py`, `app/main.py`

**`app/routers/issue_router.py`:**
```python
from fastapi import APIRouter, Depends, Query, status
from sqlalchemy.ext.asyncio import AsyncSession
from app.db.database import get_db
from app.services.issue_service import IssueService
from app.schemas.issue import IssueCreate, IssueUpdate, IssueResponse, IssuePage

router = APIRouter(prefix="/api/issues", tags=["issues"])

def get_service(db: AsyncSession = Depends(get_db)) -> IssueService:
    return IssueService(db)

@router.get("/", response_model=IssuePage)
async def list_issues(
    page: int = Query(1, ge=1, description="Page number"),
    page_size: int = Query(20, ge=1, le=100, description="Items per page"),
    service: IssueService = Depends(get_service),
):
    issues, total = await service.get_all_issues(page=page, page_size=page_size)
    return IssuePage(
        items=issues,
        total=total,
        page=page,
        page_size=page_size,
        total_pages=(total + page_size - 1) // page_size,
    )

@router.get("/{issue_id}", response_model=IssueResponse)
async def get_issue(issue_id: int, service: IssueService = Depends(get_service)):
    return await service.get_issue_or_raise(issue_id)

@router.post("/", response_model=IssueResponse, status_code=status.HTTP_201_CREATED)
async def create_issue(data: IssueCreate, service: IssueService = Depends(get_service)):
    return await service.create_issue(data)

@router.put("/{issue_id}", response_model=IssueResponse)
async def update_issue(
    issue_id: int, data: IssueUpdate, service: IssueService = Depends(get_service)
):
    return await service.update_issue(issue_id, data)

@router.delete("/{issue_id}", status_code=status.HTTP_204_NO_CONTENT)
async def delete_issue(issue_id: int, service: IssueService = Depends(get_service)):
    await service.delete_issue(issue_id)
```

`total_pages = (total + page_size - 1) // page_size` is integer ceiling division without importing `math.ceil`. All route handlers are `async def` — they `await` service calls and FastAPI handles them in the asyncio event loop without thread pool overhead.

**`app/main.py`:**
```python
from contextlib import asynccontextmanager
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from app.core.config import settings
from app.core.exceptions import AppException, app_exception_handler
from app.db.database import engine, Base
from app.api.issue_router import router as issue_router

@asynccontextmanager
async def lifespan(app: FastAPI):
    # Dev convenience — Alembic handles this in production
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
    yield
    await engine.dispose()

app = FastAPI(title="Issue Tracker API", version="1.0.0", lifespan=lifespan)

app.add_middleware(
    CORSMiddleware,
    allow_origins=settings.CORS_ORIGINS,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
app.add_exception_handler(AppException, app_exception_handler)
app.include_router(issue_router)
```

`async with engine.begin() as conn: await conn.run_sync(...)` is the async equivalent of `Base.metadata.create_all(bind=engine)`. `engine.begin()` opens an async connection; `run_sync` bridges to the synchronous DDL method. `await engine.dispose()` closes the async connection pool on shutdown.

**Run the server:**
```bash
uvicorn app.main:app --reload --port 8000
```
Test at `http://localhost:8000/docs`. Confirm `GET /api/issues` returns `IssuePage` shape.

### Commit 8: Configure Alembic Migrations

**Install:**
```bash
uv add alembic
```

- **`alembic`** — migration tool for SQLAlchemy. Tracks schema changes as versioned scripts so you can ALTER tables, roll back, and apply changes incrementally without data loss. Node.js analog: Prisma Migrate / Knex migrations.

**Why Alembic even when `create_all` exists:** `create_all` cannot ALTER existing tables. If you add a column to `Issue` in production, `create_all` silently does nothing (table already exists). Alembic generates a versioned migration script with the exact `ALTER TABLE` statement. Roll back with `alembic downgrade -1`.

**Setup:**
```bash
cd backend
alembic init alembic
```

**Edit `alembic.ini`** — find and update:
```ini
sqlalchemy.url = sqlite+aiosqlite:///./issues.db
```

**Replace `alembic/env.py` content:**
```python
import asyncio
from logging.config import fileConfig
from sqlalchemy.ext.asyncio import create_async_engine
from alembic import context
from app.db.database import Base
from app.models import issue  # noqa: F401 — importing registers Issue with Base.metadata
from app.core.config import settings

config = context.config
if config.config_file_name:
    fileConfig(config.config_file_name)

target_metadata = Base.metadata

def do_run_migrations(connection):
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()

async def run_async_migrations():
    engine = create_async_engine(settings.DATABASE_URL)
    async with engine.begin() as conn:
        await conn.run_sync(do_run_migrations)
    await engine.dispose()

def run_migrations_online():
    asyncio.run(run_async_migrations())

run_migrations_online()
```

Alembic itself is sync, but it can drive async SQLAlchemy via `conn.run_sync(do_run_migrations)`. `asyncio.run()` bridges the sync Alembic entry point into async.

**Create and apply the first migration:**
```bash
alembic revision --autogenerate -m "create issues table"
alembic upgrade head
```

`--autogenerate` inspects `Base.metadata` (all registered models) and compares to the current DB — writes the diff as a migration script in `alembic/versions/`. Review the generated file before applying.

**Daily workflow:**
```bash
# After changing a model
alembic revision --autogenerate -m "add priority to issues"
alembic upgrade head

# Roll back one step
alembic downgrade -1
```

**`core/services/issue.service.ts`:**
```typescript
import { Injectable, inject } from '@angular/core';
import { HttpClient, HttpParams } from '@angular/common/http';
import { Observable } from 'rxjs';
import { environment } from '../../../environments/environment';
import { Issue, IssuePage, CreateIssueDto, UpdateIssueDto } from '../models/issue.model';

@Injectable({ providedIn: 'root' })
export class IssueService {
  private http = inject(HttpClient);
  private apiUrl = `${environment.apiUrl}/issues`;

  getAll(page: number = 1, pageSize: number = 20): Observable<IssuePage> {
    const params = new HttpParams()
      .set('page', page)
      .set('page_size', pageSize);
    return this.http.get<IssuePage>(this.apiUrl, { params });
  }

  getById(id: number): Observable<Issue> {
    return this.http.get<Issue>(`${this.apiUrl}/${id}`);
  }

  create(data: CreateIssueDto): Observable<Issue> {
    return this.http.post<Issue>(this.apiUrl, data);
  }

  update(id: number, data: UpdateIssueDto): Observable<Issue> {
    return this.http.put<Issue>(`${this.apiUrl}/${id}`, data);
  }

  delete(id: number): Observable<void> {
    return this.http.delete<void>(`${this.apiUrl}/${id}`);
  }
}
```

Observables are **lazy** — calling `getAll()` does NOT fire an HTTP request until something subscribes (`.subscribe()` or `| async` pipe in template).

**`core/interceptors/error.interceptor.ts`:**
```typescript
import { HttpInterceptorFn, HttpErrorResponse } from '@angular/common/http';
import { catchError, throwError } from 'rxjs';

export const errorInterceptor: HttpInterceptorFn = (req, next) => {
  return next(req).pipe(
    catchError((error: HttpErrorResponse) => {
      let message = 'An unexpected error occurred';
      if (error.status === 0)   message = 'Cannot reach server — is the backend running?';
      if (error.status === 404) message = error.error?.error ?? 'Resource not found';
      if (error.status === 422) message = error.error?.error ?? 'Validation error';
      if (error.status === 500) message = 'Internal server error';
      console.error(`[HTTP ${error.status}]`, message);
      return throwError(() => new Error(message));
    })
  );
};
```

**Update `app.config.ts`:**
```typescript

  @Input({ required: true }) totalPages!: number;
  @Input({ required: true }) total!: number;
  @Output() pageChange = new EventEmitter<number>();

  get hasPrev(): boolean { return this.currentPage > 1; }
  get hasNext(): boolean { return this.currentPage < this.totalPages; }

  prev() { if (this.hasPrev) this.pageChange.emit(this.currentPage - 1); }
  next() { if (this.hasNext) this.pageChange.emit(this.currentPage + 1); }
  goTo(page: number) { this.pageChange.emit(page); }
}
```

### Commit 13: Add IssueList Container with Pagination and RxJS

**File:** `features/issues/containers/issue-list/issue-list.component.ts`

This is the primary smart component — it orchestrates state, services, and dumb components.

```typescript
import { Component, inject, OnInit } from '@angular/core';
import { Router } from '@angular/router';
import { BehaviorSubject, switchMap, distinctUntilChanged, catchError, finalize, of } from 'rxjs';
import { AsyncPipe } from '@angular/common';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
import { IssueService } from '../../../../core/services/issue.service';
import { IssuePage } from '../../../../core/models/issue.model';
import { IssueCardComponent } from '../../components/issue-card/issue-card.component';
import { ConfirmDialogComponent } from '../../components/confirm-dialog/confirm-dialog.component';
import { PaginationComponent } from '../../components/pagination/pagination.component';
import { LoadingSpinnerComponent } from '../../../../shared/components/loading-spinner/loading-spinner.component';
import { ErrorMessageComponent } from '../../../../shared/components/error-message/error-message.component';

@Component({
  selector: 'app-issue-list',
  standalone: true,
  imports: [
    AsyncPipe,
    IssueCardComponent,
    ConfirmDialogComponent,
    PaginationComponent,
    LoadingSpinnerComponent,
    ErrorMessageComponent,
  ],
  templateUrl: './issue-list.component.html',
})
export class IssueListComponent {
  private issueService = inject(IssueService);
  private router = inject(Router);

  readonly pageSize = 20;
  isLoading = false;
  error: string | null = null;
  pendingDeleteId: number | null = null;

  // BehaviorSubject as the page "source of truth"
  private page$ = new BehaviorSubject<number>(1);

  // RxJS pipeline — this is where the rubric operators live
  issuePage$ = this.page$.pipe(
    distinctUntilChanged(),                          // skip if same page emitted twice
    switchMap(page => {                              // cancel previous in-flight request
      this.isLoading = true;
      this.error = null;
      return this.issueService.getAll(page, this.pageSize).pipe(
        catchError(err => {
          this.error = err.message;
          return of(null);                           // return null so stream stays alive
        }),
        finalize(() => this.isLoading = false),      // always turn off spinner
      );
    }),
    takeUntilDestroyed(),                            // auto-unsubscribe on component destroy
  );

  get currentPage(): number { return this.page$.value; }

  onPageChange(page: number) { this.page$.next(page); }
  navigateToEdit(id: number) { this.router.navigate(['/issues/edit', id]); }
  showDeleteConfirm(id: number) { this.pendingDeleteId = id; }
  cancelDelete() { this.pendingDeleteId = null; }

  confirmDelete(id: number) {
    this.isLoading = true;
    this.issueService.delete(id).pipe(
      finalize(() => this.isLoading = false)
    ).subscribe({
      next: () => {
        this.pendingDeleteId = null;
        this.page$.next(this.currentPage);  // re-emit same page to refresh
      },
      error: err => this.error = err.message,
    });
  }
}
```

**`issue-list.component.html`:**
```html
@if (isLoading) {
  <app-loading-spinner />
}

@if (error) {
  <app-error-message [message]="error" />
}

@if (issuePage$ | async; as page) {
  @if (page && page.items.length > 0) {
    @for (issue of page.items; track issue.id) {
      <app-issue-card
        [issue]="issue"
        (editClicked)="navigateToEdit($event)"
        (deleteClicked)="showDeleteConfirm($event)"
      />
    }

    <app-pagination
      [currentPage]="page.page"
      [totalPages]="page.total_pages"
      [total]="page.total"
      (pageChange)="onPageChange($event)"
    />
  } @else if (!isLoading) {
    <p>No issues yet. <a routerLink="/issues/new">Create one</a>.</p>
  }
}

@if (pendingDeleteId !== null) {
  <app-confirm-dialog
    message="Delete this issue? This cannot be undone."
    (confirmed)="confirmDelete(pendingDeleteId!)"
    (cancelled)="cancelDelete()"
  />
}
```

`@if`, `@for` = Angular 17+ control flow (replaces `*ngIf`, `*ngFor`). `| async` subscribes to `issuePage$` and auto-unsubscribes when the component is destroyed.

`switchMap` here is the key RxJS pattern — if the user clicks page 2 before page 1 finishes loading, `switchMap` cancels the page 1 request and starts page 2. `distinctUntilChanged` prevents re-fetching when `page$.next(1)` is called but page is already 1 (e.g., after a delete on page 1).

**Verify:** `ng serve` → navigate to `/issues` → confirm list loads, pagination renders if > 20 issues.

### Commit 14: Add IssueForm Container for Create and Edit

**File:** `features/issues/containers/issue-form/issue-form.component.ts`

```typescript
import { Component, inject, OnInit } from '@angular/core';
import { FormBuilder, ReactiveFormsModule, Validators } from '@angular/forms';
import { ActivatedRoute, Router, RouterModule } from '@angular/router';
import { DestroyRef } from '@angular/core';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
import { finalize } from 'rxjs';
import { IssueService } from '../../../../core/services/issue.service';
import { IssueStatus, UpdateIssueDto, CreateIssueDto } from '../../../../core/models/issue.model';
import { LoadingSpinnerComponent } from '../../../../shared/components/loading-spinner/loading-spinner.component';
import { ErrorMessageComponent } from '../../../../shared/components/error-message/error-message.component';

@Component({
  selector: 'app-issue-form',
  standalone: true,
  imports: [ReactiveFormsModule, RouterModule, LoadingSpinnerComponent, ErrorMessageComponent],
  templateUrl: './issue-form.component.html',
})
export class IssueFormComponent implements OnInit {
  private fb = inject(FormBuilder);
  private issueService = inject(IssueService);
  private route = inject(ActivatedRoute);
  private router = inject(Router);
  private destroyRef = inject(DestroyRef);

  readonly statusOptions = Object.values(IssueStatus);
  isEditMode = false;
  issueId: number | null = null;
  isLoading = false;
  error: string | null = null;

  form = this.fb.group({
    title: ['', [Validators.required, Validators.maxLength(255)]],
    description: ['', Validators.maxLength(2000)],
    status: [IssueStatus.OPEN, Validators.required],
  });

  ngOnInit(): void {
    const id = this.route.snapshot.paramMap.get('id');
    this.issueId = id ? Number(id) : null;
    this.isEditMode = !!this.issueId;

    if (this.isEditMode) {
      this.isLoading = true;
      this.issueService.getById(this.issueId!).pipe(
        takeUntilDestroyed(this.destroyRef),   // auto-unsubscribes on destroy
        finalize(() => this.isLoading = false),
      ).subscribe({
        next: issue => this.form.patchValue(issue),
        // patchValue fills only provided fields (vs setValue which requires ALL fields)
        error: err => this.error = err.message,
      });
    }
  }

  submit(): void {
    if (this.form.invalid) return;
    this.isLoading = true;
    this.error = null;

    const data = this.form.getRawValue();
    const request$ = this.isEditMode
      ? this.issueService.update(this.issueId!, data as UpdateIssueDto)
      : this.issueService.create(data as CreateIssueDto);

    request$.pipe(
      finalize(() => this.isLoading = false)
    ).subscribe({
      next: () => this.router.navigate(['/issues']),
      error: err => this.error = err.message,
    });
  }

  get titleErrors() { return this.form.get('title')?.errors; }
  get isTitleInvalid() {
    const ctrl = this.form.get('title');
    return ctrl?.invalid && ctrl?.touched;
  }
}
```

`patchValue` fills the form with the existing issue data in edit mode. Unlike `setValue`, it doesn't require all fields — useful when the form shape differs slightly from the API response shape.

`takeUntilDestroyed(this.destroyRef)` auto-unsubscribes when the component is destroyed — if the user navigates away before the `getById` call completes, the subscription is cancelled. No `ngOnDestroy` boilerplate needed.

**Template (key parts):**
```html
<form [formGroup]="form" (ngSubmit)="submit()">
  <label>Title *</label>
  <input formControlName="title" />
  @if (isTitleInvalid) {
    @if (titleErrors?.['required']) { <span>Title is required</span> }
    @if (titleErrors?.['maxlength']) { <span>Max 255 characters</span> }
  }

  <label>Description</label>
  <textarea formControlName="description"></textarea>

  <label>Status</label>
  <select formControlName="status">
    @for (s of statusOptions; track s) {
      <option [value]="s">{{ s }}</option>
    }
  </select>

  @if (error) { <app-error-message [message]="error" /> }

  <button type="button" routerLink="/issues">Cancel</button>
  <button type="submit" [disabled]="form.invalid || isLoading">
    {{ isEditMode ? 'Update' : 'Create' }}
  </button>
</form>
```

**Verify:** Create a new issue → redirects to list. Edit an existing issue → form pre-populated → save updates the record.

## React → Angular Concept Map

| React | Angular | Notes |
|---|---|---|
| Function component | `@Component` class | Components are classes in Angular |
| `useState` | `signal()` or class property | Signals = Angular 16+ reactive primitive |
| `useEffect` | `ngOnInit()` / `ngOnDestroy()` | Lifecycle hooks as class methods |
| Custom hook | Angular Service | Singleton, injected via DI |
| Context Provider | `providedIn: 'root'` | No wrapper needed |
| Props | `@Input()` | Property binding: `[prop]="value"` |
| Callback prop | `@Output()` + `EventEmitter` | Event binding: `(event)="handler($event)"` |
| Axios instance | `HttpClient` | Angular's built-in HTTP client |
| Axios interceptor | `HttpInterceptorFn` | Registered in `app.config.ts` |
| React Router | `@angular/router` | `app.routes.ts` |
| Zod / controlled form | Reactive Forms + `Validators` | `FormBuilder.group({})` |
| Promise | `Observable` (RxJS) | Lazy, composable, cancellable |
| `React.lazy()` | `loadComponent: () => import(...)` | Lazy route loading |
| AbortController | `switchMap` | Cancels in-flight request on new emission |
| useEffect cleanup | `takeUntilDestroyed` | Auto-unsubscribe on component destroy |
